  • Headings: Use built-in Heading styles (Home ‚Üí Styles), not bold text
  • Alt text: Right-click images ‚Üí Edit Alt Text ‚Üí Write description

üìä PowerPoint Accessibility Checklist

Run Accessibility Checker: Review ‚Üí Check Accessibility

  • Slide titles: Every slide needs a unique title (use Layout with title placeholder)
  • Reading order: Home ‚Üí Arrange ‚Üí Selection Pane ‚Üí Drag to reorder
  • Alt text: Right-click images/charts ‚Üí Edit Alt Text
  • Links: Insert ‚Üí Link ‚Üí Display text (make it descriptive)
  • Tables: Simple tables only; use "Header Row" in Table Design
  • Color contrast: 4.5:1 for text; avoid red/green combinations
  • Animations: Keep simple; avoid auto-advance; no flashing
  • Videos: Add captions (Insert ‚Üí Video ‚Üí right-click ‚Üí Insert Captions)

üñºÔ∏è Alt Text Quick Guide

Image type Alt text approach
Informative Describe the content and purpose. "Student studying in library with laptop"
Decorative Mark as decorative or use alt="" (empty). Don't describe.
Functional (button/link) Describe the action. "Search", "Submit form", "Go to homepage"
Chart/Graph Summarize the data and trend. Provide data table nearby.
Complex (diagram) Brief alt + link to full description or long description nearby.
Text in image Include all visible text. Better: use real text instead.
Logo "University of Arizona logo" or just "University of Arizona"

Tips:

  • Be concise (usually under 125 characters)
  • Don't start with "Image of..." or "Picture of..."
  • Context matters‚Äîsame image may need different alt text in different places

üîç 5-Minute Accessibility Audit

  • Tab through the page ‚Äî Can you reach everything? Is focus visible?
  • Zoom to 200% ‚Äî Does content reflow without horizontal scrolling?
  • Check headings ‚Äî Run WAVE or axe; is there a logical H1‚ÜíH2‚ÜíH3 structure?
  • View without images ‚Äî Are alt texts meaningful? Is info still clear?
  • Test one form ‚Äî Are labels connected? Do errors identify the problem?
  • Play any video ‚Äî Are captions available and accurate?
  • Check links ‚Äî Do link texts make sense out of context? (No "click here")
  • Try high contrast ‚Äî Does the page still work in Windows High Contrast mode?

Found issues? Log them and prioritize by user impact.

‚úâÔ∏è Accessible Email Checklist (Outlook)

  • Subject line: Clear and descriptive (not "Newsletter" alone)
  • Headings: Use Format Text ‚Üí Styles (not just bold text)
  • Images: Right-click ‚Üí Edit Alt Text (or mark decorative)
  • Links: Descriptive text, not bare URLs or "click here"
  • Lists: Use built-in bullet/numbered lists
  • Tables: Simple structure only; use for data, not layout
  • Color: Don't rely on color alone; maintain contrast
  • Attachments: Include accessible versions; mention format in email
  • Plain text: Provide plain text alternative for complex HTML emails
  • Test: Send yourself a test; check in different email clients

Run check: Options ‚Üí Check Accessibility (before sending)

üé¨ Video & Media Accessibility

Content type Requirements
Pre-recorded video Captions (synchronized text for dialogue + sounds)
Pre-recorded audio Transcript (text version of all audio content)
Live video/audio Live captions (CART) for real-time events
Visual-only content Audio description or text description

Caption quality targets:

  • 99%+ accuracy for educational content
  • Speaker identification for multiple speakers
  • Sound effects noted: [applause], [music playing]
  • Synchronized within 2-3 seconds of audio

Tools: Panopto (auto-caption + edit), YouTube Studio, Rev.com
