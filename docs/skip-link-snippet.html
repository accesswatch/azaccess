Skip Link Code Snippet

Ready-to-use skip navigation link code for your websites.

Why Skip Links?

Skip links allow keyboard and screen reader users to bypass repetitive navigation and jump directly to main content. They are:

  • Required for WCAG 2.4.1 Bypass Blocks (Level A)
  • Essential for keyboard users who would otherwise tab through every nav link
  • Helpful for screen reader users to quickly reach content
  • Usually hidden until focused, so they don't affect visual design

HTML Code

Add this as the first element inside your <body> tag:

<!-- Skip link - place as first element in body -->
<a class="skip-link" href="#maincontent">Skip to main content</a>

<!-- Your header/navigation here -->
<header>...</header>

<!-- Main content with matching ID -->
<main id="maincontent">
  ...
</main>

CSS Code

Add this CSS to hide the skip link until it receives keyboard focus:

/* Skip link - hidden until focused */
.skip-link {
  position: absolute;
  left: -9999px;
  top: auto;
  width: 1px;
  height: 1px;
  overflow: hidden;
  z-index: 100;
}

/* Visible when focused */
.skip-link:focus {
  position: fixed;
  top: 0;
  left: 0;
  width: auto;
  height: auto;
  overflow: visible;
  padding: 0.75rem 1.5rem;
  background: #0C234B;  /* UA Arizona Blue */
  color: #ffffff;
  font-size: 1rem;
  font-weight: 600;
  text-decoration: none;
  outline: 3px solid #AB0520;  /* UA Arizona Red */
  outline-offset: 2px;
}

Variations

Multiple Skip Links

For complex pages, you can offer multiple targets:

<nav class="skip-links" aria-label="Skip navigation">
  <a href="#maincontent">Skip to main content</a>
  <a href="#search">Skip to search</a>
  <a href="#footer">Skip to footer</a>
</nav>

With Smooth Scroll

Add smooth scrolling (but ensure it respects reduced motion preferences):

html {
  scroll-behavior: smooth;
}

@media (prefers-reduced-motion: reduce) {
  html {
    scroll-behavior: auto;
  }
}
