Accessible Alert Banners

How to create accessible notification and alert banners for your website.

Reading time: 8 minutes

Alert Accessibility Overview

Alerts communicate important information to users. Making them accessible ensures everyone receives critical updates.

Key Requirements

  • ARIA roles:Userole="alert"orrole="status"for dynamic announcements
  • Color + text:Don't rely on color alone—use icons and text labels
  • Contrast:Text must have 4.5:1 contrast ratio
  • Focus management:Critical alerts may need to receive focus
  • Dismissible:Allow users to close non-critical alerts

Alert Examples

Static Alerts

Alerts that are present when the page loads:

Information

Registration opens on January 15.View schedule

Warning

Code Patterns

Basic Static Alert

For alerts present on page load, no ARIA role is needed:

<div class="alert alert-info">
 <strong> Information:</strong>
 Registration opens on January 15.
</div>

Dynamic Alert (Appears After Action)

Userole="alert"for urgent messages that appear dynamically:

<div role="alert" class="alert alert-danger">
 <strong> Error:</strong>
 Form submission failed. Please check required fields.
</div>

role="alert" vs role="status"

  • role="alert":Interrupts screen readers immediately (use sparingly)
  • role="status":Announced at next pause, less intrusive

Dismissible Alert with ARIA

<div class="alert alert-warning alert-dismissible" role="alert">
 <strong> Warning:</strong>
 Your session will expire in 5 minutes.
 
 <button type="button" 
 class="alert-dismiss" 
 aria-label="Dismiss alert">
 &times;
 </button>
</div>

<script>
document.querySelector('.alert-dismiss').addEventListener('click', function() {
 const alert = this.closest('.alert');
 alert.remove();
 // Return focus to a logical location
});
</script>

Live Region for Updates

For status updates that don't require immediate attention:

<!-- Hidden live region -->
<div id="status-region" 
 role="status" 
 aria-live="polite" 
 class="visually-hidden">
</div>

<script>
function announceStatus(message) {
 const region = document.getElementById('status-region');
 region.textContent = message;
}

// Usage
announceStatus('3 items added to cart');
</script>

Accessible CSS

/* Base alert styles */
.alert {
 padding: 1rem 1.25rem;
 border-radius: 8px;
 border-left: 4px solid;
 margin-bottom: 1rem;
}

/* Use both color AND icon/text for meaning */
.alert-info {
 background: #cce5ff;
 border-color: #004085;
 color: #004085; /* 4.5:1 contrast on background */
}

.alert-success {
 background: #d4edda;
 border-color: #155724;
 color: #155724;
}

.alert-warning {
 background: #fff3cd;
 border-color: #856404;
 color: #856404;
}

.alert-danger {
 background: #f8d7da;
 border-color: #721c24;
 color: #721c24;
}

/* Links within alerts */
.alert a {
 color: inherit;
 text-decoration: underline;
 font-weight: 600;
}

/* Dismissible alerts */
.alert-dismissible {
 position: relative;
 padding-right: 3rem;
}

.alert-dismiss {
 position: absolute;
 right: 0.75rem;
 top: 50%;
 transform: translateY(-50%);
 background: transparent;
 border: none;
 font-size: 1.5rem;
 cursor: pointer;
 color: inherit;
 padding: 0.5rem;
}

.alert-dismiss:focus {
 outline: 2px solid currentColor;
 outline-offset: 2px;
}

Best Practices

Do

  • Use icons AND text labels (not just color)
  • Keep alert messages concise and actionable
  • Provide a way to dismiss non-critical alerts
  • Test with screen readers to verify announcements
  • Userole="alert"sparingly—only for truly urgent messages
  • Return focus appropriately after dismissing an alert

Don't

  • Don't rely on color alone to convey alert type
  • Don't userole="alert"for page content present on load
  • Don't auto-dismiss alerts too quickly (minimum 5 seconds for non-critical)
  • Don't trap focus inside alert banners
  • Don't use alerts for decorative purposes

Testing Alerts

  1. Visual check:Can you understand the alert type without color?
  2. Contrast:Test text against background (4.5:1 minimum)
  3. Keyboard:Can dismiss button be reached and activated with keyboard?
  4. Screen reader:Is alert announced when it appears?
  5. Focus:After dismissing, does focus go to a logical location?
