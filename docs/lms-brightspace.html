Brightspace Accessibility Checklist

Make your Brightspace (D2L) courses accessible to all students with this comprehensive guide.

Course Home Setup

Course Image

  • Add alt text describing the course image
  • Avoid text in images (it can't be read by screen readers)

Course Navigation

  • Use clear, descriptive names for navigation links
  • Keep navigation consistent throughout the course
  • Limit top-level items to 7-9 for easier navigation

Announcements

  • Use descriptive subject lines
  • Structure with headings if longer than a paragraph
  • Provide text alternatives for any images

Content Modules

Module Organization

  • Use descriptive module titles (not "Week 1" alone‚Äîadd the topic)
  • Maintain consistent structure across modules
  • Start each module with learning objectives

HTML Content Pages

When creating content in the Brightspace HTML editor:

  • Use the Format dropdown to apply heading styles (Heading 2, Heading 3, etc.)
  • Never use Heading 1 ‚Äî Brightspace adds the page title as H1
  • Add alt text to images ‚Äî Click the image ‚Üí Properties ‚Üí Alternative Text
  • Create real lists ‚Äî Use the numbered/bulleted list buttons
  • Make links descriptive ‚Äî "Read the syllabus" not "Click here"

üí° Using the Accessibility Checker

In the HTML editor toolbar, click the Accessibility button (checkmark icon). It will flag issues like missing alt text and empty headings.

Uploaded Files

  • Word documents: Run Review ‚Üí Check Accessibility before uploading
  • PDFs: Ensure they're tagged and have alt text (see PDF Remediation Guide)
  • PowerPoints: Use slide layouts and add alt text to images
  • Provide HTML/text alternatives when possible

Multimedia Content

Videos (Panopto, YouTube, Kaltura)

  • Captions are required for all videos (WCAG 1.2.2)
  • Auto-captions are a starting point but must be edited for accuracy
  • Aim for 99% caption accuracy
  • Include speaker identification in captions

See Panopto Accessibility Guide ‚Üí

Audio Content

  • Provide a text transcript for all audio files
  • Transcripts should include speaker names and relevant sounds

Embedded Content

  • Check that embedded tools (H5P, third-party) are keyboard accessible
  • Provide alternatives if embedded content isn't accessible

Quizzes & Assessments

Quiz Settings

  • Extended time: Use "Special Access" to grant extra time to students with accommodations
  • Multiple attempts: Consider allowing retakes to reduce anxiety
  • Clear instructions: Explain format, time limits, and allowed resources at the start

Question Accessibility

  • Avoid images of text in questions
  • Add alt text to any images used in questions
  • Ensure math equations use MathML or LaTeX (not images)
  • Make sure answer choices are clearly distinguishable

Accommodations Workflow

  1. Student registers with Disability Resource Center (DRC)
  2. DRC sends accommodation letter to instructor
  3. In Brightspace: Quizzes ‚Üí [Quiz Name] ‚Üí Restrictions ‚Üí Special Access
  4. Add the student and set extended time multiplier (e.g., 1.5x, 2x)

Discussions

  • Use descriptive topic titles
  • Set clear formatting expectations in instructions
  • Model accessible posting (headings, lists, alt text)
  • Allow multiple response formats (text, audio, video with captions)

Assignments

  • Provide clear, structured instructions
  • Accept multiple file formats when possible
  • If using rubrics, ensure criteria are clearly written
  • Consider accessibility when requiring specific tools
