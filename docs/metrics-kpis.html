Why Measure Accessibility

Effective accessibility programs require data-driven management:

  • Track progress:Demonstrate improvement over time
  • Identify gaps:Find areas needing attention
  • Allocate resources:Direct effort where it's most needed
  • Report to leadership:Communicate status and ROI
  • Ensure compliance:Document meeting regulatory requirements

Metric Categories

Compliance

Technical conformance with WCAG standards

Operational

Service delivery and response times

Training

Staff education and awareness

Vendor accessibility compliance

Compliance Metrics

Website Accessibility Score

Metric Description Target
Overall Compliance Rate Percentage of pages passing automated WCAG 2.1 AA checks 95%
Critical Issue Count Number of Level A failures per 100 pages <5
High-Traffic Page Score Compliance of top 100 visited pages 100%
New Content Score Compliance of content published in last 30 days 98%

Issue Severity Breakdown

Severity Definition Example Issues
Critical Blocks access for users with disabilities Missing form labels, no keyboard access, missing alt text
Serious Creates significant barriers Poor contrast, missing headings, non-descriptive links
Moderate Causes difficulty but workarounds exist Inconsistent navigation, missing skip links
Minor Best practice violations Missing lang attribute, redundant alt text

Reporting and Dashboards

Dashboard Components

The accessibility dashboard should include:

  • Executive Summary:Overall compliance score with trend
  • Issue Breakdown:Issues by severity and type
  • Unit Comparison:Compliance by college/department
  • Trend Charts:Progress over time
  • Action Items:Open issues requiring attention

Report Types

Report Audience Frequency Key Metrics
Executive Dashboard Leadership Monthly Overall score, trends, risks
Steering Committee Committee members Monthly All KPIs, action items
Unit Report Unit liaisons Quarterly Unit-specific metrics
Annual Report University leadership Annual Year-over-year progress

Data Sources

  • Automated scanning:SiteImprove, WAVE, axe
  • Manual audits:Accessibility team assessments
  • Feedback system:Accessibility complaint tracking
  • Training LMS:Completion records
  • Procurement system:VPAT reviews
