Accessibility Testing Tools

A comprehensive directory of tools for testing web, document, and mobile accessibility.

Reading time: 8 minutes

Testing Overview

Accessibility testing requiresboth automated and manual testing. Automated tools catch about 30-40% of issues; the rest require human judgment.

Automated Testing

Catches: missing alt text, color contrast, missing labels, invalid ARIA, heading structure

Manual Testing

Catches: keyboard navigation, screen reader experience, focus management, meaningful alt text

Browser Extensions

Install these in Chrome, Firefox, or Edge for quick testing.

Tool Best For Cost
axe DevTools Comprehensive automated scanning; developer-friendly; integrates with DevTools Free (Pro available)
WAVE Visual overlay showing issues in context; great for beginners Free
Accessibility Insights Guided manual testing; FastPass for quick checks; detailed reports Free (Microsoft)
ARC Toolkit Professional-grade; WCAG mapping; detailed technical reports Free
Lighthouse Built into Chrome DevTools; includes accessibility audit Free (built-in)

Recommended Setup

Installaxe DevToolsfor automated testing andAccessibility Insightsfor guided manual testing. Together they cover most needs.

Quick Screen Reader Test

  1. Turn on the screen reader
  2. Close your eyes or look away
  3. Navigate through the page using only keyboard
  4. Can you understand the content? Complete tasks?

Automated Testing in CI/CD

Integrate accessibility testing into your development pipeline.

Tool Integration
axe-core JavaScript library; works with any test framework
Pa11y Command-line tool; CI-friendly; generates reports
Playwright Built-in accessibility testing with axe integration
Lighthouse CI Run Lighthouse audits in CI; track scores over time

Example: axe-core with Jest

import { axe, toHaveNoViolations } from 'jest-axe';

expect.extend(toHaveNoViolations);

test('page has no accessibility violations', async () =>{
 const { container } = render(<MyComponent />);
 const results = await axe(container);
 expect(results).toHaveNoViolations();
});

Example: Pa11y CLI

# Test a single URL
pa11y https://example.com

# Test with specific standard
pa11y --standard WCAG2AA https://example.com

# Output as JSON for CI
pa11y --reporter json https://example.com

Recommended Testing Process

  1. Automated scan— Run axe DevTools or WAVE first
  2. Fix automated issues— Address all flagged problems
  3. Keyboard test— Tab through entire page without mouse
  4. Screen reader test— Navigate with NVDA or VoiceOver
  5. Zoom test— Zoom to 200% and check layout
  6. Color test— Check contrast; simulate color blindness
  7. Content review— Check alt text quality, link text, headings
