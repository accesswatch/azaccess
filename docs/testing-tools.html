Accessibility Testing Toolkit

Demo summary of the shared testing-tools guidance.

Testing layering strategy

  1. Review designs against WCAG 2.2 AA before development.
  2. Run automated scans in CI/CD for every build.
  3. Perform keyboard-only testing for key flows.
  4. Validate critical scenarios with screen readers on key platforms.
  5. Expand to additional assistive technologies for high-impact services.

Tool catalog

Tool Purpose Notes
Accessibility Insights FastPass automated + guided manual checks. Align results with WCAG mapping.
axe DevTools / Pa11y CI Automated rules engine in CI/CD. Fail builds on critical issues.
WAVE Visual overlay for quick content audits. Spot color contrast and heading patterns.
Screen readers Manual validation of semantics. NVDA, VoiceOver, TalkBack for critical flows.
Colour Contrast Analyser Verify text and UI contrast ratios. Required for custom palettes.
